# 钉钉群消息推送机器人项目总结

## 项目概述

本项目是一个完整的钉钉群消息推送机器人系统，集成了前端React.js应用和后端Node.js + Express服务，实现了定时任务管理和消息推送功能。

## 技术实现

### 前端技术栈
- React.js (TypeScript)
- React Router v6 (页面路由)
- react-toastify (消息提示)
- CSS Modules (样式管理)

### 后端技术栈
- Node.js
- Express.js
- axios (HTTP客户端)
- cors (跨域处理)

### 核心功能实现

#### 1. 全局导航
- 顶部导航栏包含"定时任务"、"调试中心"、"设置"三个Tab
- 默认选中"定时任务"页面
- 使用React Router实现页面切换

#### 2. 定时任务页
- 展示四种消息模板预览
- 显示发送日志区域（占位符）

#### 3. 调试中心页
- 提供四种消息类型选择卡片
- 根据选择的消息类型动态显示发送按钮文案
- 集成react-toastify显示发送结果反馈
- 展示最近5条发送记录（时间倒序）

#### 4. 设置页
- 钉钉机器人Webhook配置
- 群名称设置
- 定时任务时间配置

#### 5. 后端API
- `/api/send-message` (POST): 发送钉钉消息
- `/api/debug-records` (GET): 获取调试记录

#### 6. 数据存储
- 使用内存数组存储调试记录
- 限制记录数量为最近5条

## 运行说明

### 开发环境运行
```bash
# 启动后端服务
cd server && npm run dev

# 启动前端应用
cd client && npm start

# 或者在根目录同时启动前后端
npm run dev
```

### 生产环境部署
```bash
# 构建前端应用
cd client && npm run build

# 启动后端服务
cd server && npm start
```

## 扩展建议

1. **数据库集成**: 使用MongoDB或PostgreSQL替代内存存储
2. **定时任务**: 集成node-cron实现真正的定时消息推送
3. **权限管理**: 添加用户认证和权限控制
4. **多群组支持**: 支持向多个钉钉群发送消息
5. **消息模板管理**: 支持自定义消息模板
6. **发送日志**: 完善发送日志查看功能

## 注意事项

1. 钉钉机器人Webhook需要配置关键词才能发送消息
2. 调试记录仅保存在内存中，重启服务后会丢失
3. 前后端需要在同一域名下部署以避免跨域问题
